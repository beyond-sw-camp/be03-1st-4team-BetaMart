# 전시(USER) 시스템 Ideation

1. 회원 테이블(USER TABLE)
    - 이름(NAME)
    - ID
    - PW
    - email

2. 주소 테이블 - B마트 배달 도착 예상시간은 B마트주소와 회원 현재 선택 주소 사이 거리의 걸리는 시간을 구해야 하는데
                지도를 활용하는 것이 아니면 주소 사이의 걸리는 시간을 구하는 것은 어려울 것 같아서 추가적으로 생각 필요.
    - ID(회원 테이블)
    - 주소
    - 회원의 현재 선택 주소 (O,x)

3. 인기 상품 테이블 - 종류별 상관없이 나열
    - 상품이름
    - 가격
    - 평점
    - 상품 설명

4. 특가 상품(세일) - 종류별로 특가상품 테이블 따로 만들지 안만들지 고민
    - 상품이름
    - 가격
    - 세일률 %
    - 평점
    - 상품 설명

5. 신상품 - 입점 2주내의 상품들 - 종류별 상관없이 나열 or (이번주 신상, 맛있는것, 편리한것)
    - 상품이름
    - 가격
    - 신상품 세일률 %
    - 평점
    - 상품 설명

6. 장바구니
    - 상품이름(상품이 있는 테이블에 의존(FK))
    - 갯수
    - 선택상품가격X갯수(해당 상품 가격 X 갯수 값)
    + (테스트케이스) 장바구니의 총 가격을 구할때 선택상품가격X갯수의 총합을 구해서 표시.
